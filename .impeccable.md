# Design Context — DirectiveFilms

This file captures the persistent design context for the project. Read it first before designing any UI work.

The **brand** is already encoded in `src/globals.css`, `src/tokens.js`, and the `/design-system` page — that is the source of truth for colors, type, spacing, and motion. This document captures the *intent* behind those decisions and the design context for new surfaces (especially the admin/CMS).

---

## Design Context

### Users

**Public site visitors** are prospects evaluating DirectiveFilms for strategic video work. They arrive expecting a film studio that takes craft seriously. They scan first, read second — the page has to *look* like the work is good before they'll read a word.

**Admin/CMS users** are non-technical: a founder or marketer who needs to edit copy, swap an image, add a project, publish a testimonial. They are not developers. They do not want to see "fields", JSON, schemas, or any tool-coded vocabulary. Editing the site should feel like editing the live page itself — close to Google Docs / Webflow Editor mode, not WordPress wp-admin.

### Brand Personality

Cinematic. Confident. Premium with restraint.

Voice: considered, direct, no em-dashes, no marketing fluff (per recent commit `3fa0a33`). The tone is "Meet the Team" not "About Us" — friendly, but tight.

Emotional goals on the public site: quiet authority, taste, the sense that the studio will *get it right*.

Emotional goals on the admin: calm, approachable, in control. The editor should never feel they're about to break something.

### Aesthetic Direction

**Palette** — Signature gold (`#FDB714`) used surgically as an accent over deep black (`#0d0d0d` / `#111111`) and crisp white. The site moved away from navy-blue toward neutral gray/black for a more cinematic posture (commit `b33ad65`). Gold never floods — it punctuates.

**Typography** — Inter for both display and body (the codebase declares Playfair in CLAUDE.md but the actual `--font-display` is Inter; the project converged on a clean sans-serif posture). Space Mono in uppercase with `0.2em` letter-spacing for labels. Tight `-0.02em` tracking on display sizes. This combination reads as editorial / cinema-agency, not corporate SaaS.

**Layout** — Generous. 1400px max width, 60px section padding, 40px container padding. Lots of room for the work to breathe. Pill buttons (`--radius-pill: 30px`), low-alpha neutral shadows, gold glow shadows reserved for emphasis moments.

**Motion** — Framer Motion + Lenis smooth scroll. Premium cubic-bezier `(0.25, 0.1, 0.25, 1)`. Spring presets for interactive feedback. `prefers-reduced-motion` already respected throughout. The motion bar is set to "award-winning" (per `AWARD-WINNING-ANIMATIONS.md`) — animations are intentional, never decorative.

**References:**
- Public site direction: a24 / MUBI editorial restraint, agency-cinematic posture.
- Admin/CMS direction: **Webflow Editor mode** — site renders as live, content becomes editable on click, minimal floating chrome. Approachable, not dev-tool-coded.

**Anti-references:**
- Generic SaaS dashboards (Material/Bootstrap-y, soulless).
- WordPress wp-admin (cluttered sidebars, plugin-induced complexity).
- Loud or overdesigned admin chrome that competes with the site content.
- Dev-tool-coded interfaces (raw JSON, exposed schema field names, "Add document" terminology).

### Admin / CMS Specific Direction

The admin is an *extension of the brand*, not a separate product. It must inherit the brand's typography and palette and feel of a piece with the public site.

- **Posture**: Webflow Editor mode. The live site is the editor. The editor moment beats any dashboard.
- **Chrome**: minimal. A discreet bottom bar for global actions (publish, history, account). A collapsible left rail for collections (Projects, Team, Services, Testimonials). Everything else lives on the page itself.
- **Interaction**: hover reveals editability subtly (thin outline + a small affordance icon). Click commits to editing. Edits are inline wherever the content type allows (text, images). Structured collections use a focused side panel rather than a modal.
- **Theme**: support **both light and dark** with a user toggle. Light mirrors the public site exactly. Dark is a deeper, calmer cinematic version — think Premiere/Final Cut posture — with the gold accent surviving on dark surfaces.
- **Vocabulary**: never use "document", "field", "schema", "draft", "mutation", "type". Use "page", "headline", "photo", "project", "save", "publish". Editor-first language.
- **Visual feedback**: every action shows what changed. Saves are autosaved or near-autosaved. The "3 changes pending" affordance lives in the bottom bar.

### Accessibility & Inclusion

- WCAG AA target for contrast across light and dark modes (the gold on white needs special care — `#FDB714` only meets AA at large sizes on white, so gold-on-white text below 18px should be reserved for non-essential decoration).
- `prefers-reduced-motion` is already honored in `globals.css` — preserve this behavior in any new motion work.
- Keyboard nav must work everywhere in the admin (Webflow-class editors get judged on this).
- Touch targets: 44×44px minimum for any tap target.

### Design Principles

1. **The site is the editor.** Never substitute a dashboard for the page itself. Editing happens where the content lives.
2. **Surgical gold.** Gold is the only accent. Use it for one thing per screen — the action that matters most.
3. **Calm density.** Generous spacing on the public site, calm clarity on the admin. Never crowded, never sparse to the point of feeling unfinished.
4. **Editor-first language.** "Headline", not "field". "Project", not "document". "Publish", not "deploy".
5. **Motion as intent.** Motion communicates state and structure, never decoration. If a transition doesn't say something, cut it.
6. **Continuity over novelty.** Admin inherits brand. New surfaces extend the system in `globals.css`; they don't invent new tokens.
